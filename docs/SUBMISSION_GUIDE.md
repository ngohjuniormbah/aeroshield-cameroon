# Submission Guide

This document translates the hackathon requirements into a clean submission checklist for your team.

## What the competition expects
Your final submission should include:
- source code repository or notebook
- dashboard access link
- pitch deck PDF
- demo video link

## What this project already gives you
This project package gives you:
- full training pipeline
- full backend API
- full Streamlit dashboard
- project explanation and judging narrative
- submission guidance

## What you still need to produce before final submission

### 1. GitHub repository
Upload this whole project to GitHub.
Suggested repo name:
`aeroshield-cameroon`

### 2. Dashboard deployment link
Deploy either:
- Streamlit dashboard
- or FastAPI + Streamlit together on a cloud host

Examples:
- Streamlit Community Cloud for dashboard
- Render or Railway for API/backend
- Hugging Face Spaces if you want a public demo

### 3. Pitch deck PDF
Prepare 5 to 7 slides.
Recommended slide order:
1. Problem and urgency
2. Our solution: AeroShield Cameroon
3. Data and AI pipeline
4. Dashboard and demo screens
5. Impact in Cameroon
6. Why it is scalable and innovative
7. Closing ask / vision

### 4. Demo video
Record a 3-minute video that shows:
- the problem
- the model idea
- the map and dashboard
- one city case example
- why this is useful

## Best demo flow

### Minute 1
Explain the problem and why physical sensor coverage is hard.

### Minute 2
Show the dashboard map, city table, and city history.

### Minute 3
Explain the impact, scalability, and future upgrade path with real PM2.5 data.

## Final checklist
- [ ] Team is 2 to 4 members
- [ ] Team has gender diversity
- [ ] Code pushed to GitHub
- [ ] Model trains successfully
- [ ] Dashboard runs successfully
- [ ] Pitch deck exported as PDF
- [ ] Demo video under 3 minutes
- [ ] Final submission links collected in one document

## Suggested submission text

**Project Title:** AeroShield Cameroon

**Short Description:**
AeroShield Cameroon is a climate-driven AI platform that predicts next-day city-level air-quality risk from meteorological data. It acts as a virtual sensor network and provides map-based alerts, climate-vs-risk analysis, and decision support for climate and health resilience.
